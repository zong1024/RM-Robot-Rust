# 控制安全加固实施计划

## 目标

在不更换运行时和硬件抽象层的前提下，使控制输出对陈旧反馈、未标定云台、异常 CAN 帧、
调度漏拍和主循环卡死默认失效安全。

## 实施顺序

- [x] 为反馈健康、绝对角度和调度漏拍编写失败回归测试。
- [x] 增加电机短超时、帧进度跟踪和真实反馈时间间隔。
- [x] 增加云台标定配置与绝对角度换算，默认禁止云台输出。
- [x] 向整车控制器传入真实 `dt`，漏拍时锁零并记录诊断。
- [x] 收紧 CAN 帧校验、硬件过滤器和单次中断处理预算。
- [x] 启用独立看门狗，并只在完整控制周期后喂狗。
- [x] 扩展 CI 检查，固定 Rust 版本，增加许可证与部署说明。
- [x] 运行格式、测试、Clippy、ARM 构建并审查最终差异。
