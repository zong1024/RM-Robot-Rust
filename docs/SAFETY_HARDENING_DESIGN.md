# 控制安全加固设计

## 目标

修复反馈陈旧、云台未标定、控制循环漏拍、CAN 输入过宽和主循环卡死时缺少恢复机制的问题，
同时保持现有 `domain -> subsystem -> app -> platform` 模块边界。

## 安全策略

- 遥控超时保持 100 ms；电机控制使用更短的 20 ms 超时。
- 电机反馈必须持续更新。控制器只在新反馈到达时更新基于编码器差分的状态。
- 云台默认未标定。未显式配置机械零点、方向和角度范围时，云台始终输出零电流，
  但不阻止底盘工作。
- 云台角度由绝对编码器零点和方向计算，不再把上电位置当机械零点。
- 主循环使用真实经过时间。漏拍超过安全阈值时，本周期输出归零并重置控制器。
- CAN 仅接受配置中的标准数据帧，且要求 DLC 为 8；每次中断限制处理帧数。
- 独立看门狗仅在完整控制周期结束后喂狗，主循环卡死时触发复位。

## 验证

- 主机单元测试覆盖反馈超时、帧进度、绝对角度、未标定锁零和漏拍锁零。
- CI 同时运行库测试、主机 Clippy、ARM binary Clippy 和 ARM release 构建。
- 实机验证必须在架空状态下完成，并先配置云台标定参数。
