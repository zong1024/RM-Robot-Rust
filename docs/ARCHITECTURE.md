# 架构说明

## 设计目标

本工程把“设备通信”和“机器人控制”彻底分开：

- 更换 IMU 不影响底盘和云台 PID。
- 增加视觉、里程计融合或世界坐标控制时，不需要修改 CAN 中断。
- 纯逻辑模块可在 x86_64 上执行单元测试。
- 固定周期、无堆分配、定长数组，适合 STM32F407 的实时约束。

## 数据流

```text
USART3 中断 -> S.BUS 解码 -> RemoteData
CAN1 中断   -> 四个 M3508 -> MotorFeedback[4]
CAN2 中断   -> 6623/6020  -> MotorFeedback
                                  |
                                  v
                        RobotController (1 kHz)
                         /       |        \
                  Remote     Chassis     Gimbal
                    |           |           |
                    +------ 安全门 ----------+
                                |
                      CAN1 0x200 / CAN2 0x1FF
```

## 模块边界

### `domain`

只定义数据和协议：

- `command.rs`：底盘、云台和整车命令。
- `motor.rs`：统一电机反馈与连续编码器跟踪。
- `remote.rs`：S.BUS 解码、通道映射、解锁状态机。
- `can_protocol.rs`：标准电机与 6623 的帧编解码。

### `chassis`

- `kinematics.rs`：普通轮差速与麦克纳姆全向运动学。
- `controller.rs`：轮型切换保护和四个独立 M3508 速度环。
- `mod.rs`：底盘子系统的公开接口。

未来的底盘运动学、跟随模式、功率限制和标定应继续放在本目录中。

`WheelMode` 由遥控器 SwB 产生并随 `ChassisCommand` 传递。模式变化时控制器
先清空四个 PID 并输出一周期零电流，然后才应用新运动学。普通轮模式使用
前进/转向两个自由度；麦克纳姆模式使用前进/横移两个自由度，右摇杆仍完整保留给云台。

### `gimbal`

- `controller.rs`：6623/GM6020 从位置环到速度环的级联控制。
- `mod.rs`：云台子系统的公开接口。

偏航 6623 没有直接转速反馈，因此先由编码器差分得到速度，再进行低通滤波。
俯仰 GM6020 直接使用反馈转速。

未来的世界坐标控制、自动瞄准目标管理和机械标定应继续放在本目录中。

### `control`

- `pid.rs`：带积分和输出限幅的固定周期 PID。

该目录只保存被底盘和云台共同复用的基础控制算法，不承载具体机器人子系统。

### `estimation`

- `attitude.rs` 定义 `AttitudeProvider`。后续 IMU 驱动只需实现该 trait。
- `odometry.rs` 根据当前轮型和四轮转速计算前向、侧向、旋转速度及二维位姿。
- 当外部姿态有效时，里程计使用外部 yaw；否则使用左右轮差速积分 yaw。

未来可增加：

- `Bmi088Attitude`：板载或外置 BMI088。
- Mahony、Madgwick 或 EKF 姿态融合。
- 轮速 + IMU + 视觉的二维/三维 EKF。
- 云台世界坐标角控制。

### `app`

`RobotController` 是唯一整车编排入口。输入是 `RobotSensors`，输出是
`RobotOutput`。它不依赖 STM32 PAC，因此可直接仿真或离线回放。

### `platform`

只存在于固件二进制：

- 初始化 CAN1/CAN2 和过滤器。
- 在中断中解析定长 CAN 帧。
- 初始化 USART3 并在中断中解析 S.BUS。
- 提供快照和定长发送函数。

## 实时性

- SysTick 产生 1 ms 调度节拍。
- CAN 与 USART 中断不运行控制算法。
- 主循环每个节拍只执行定长工作，不包含动态分配和无界遍历。
- CAN 发送邮箱忙时记录计数并丢弃本周期命令，下一周期重新发送最新命令。
- 设备新鲜度统一按 100 ms 判断。

如果后续加入耗时较大的滤波或通信，建议迁移到 RTIC/Embassy，并保持当前模块边界。

## 新功能归属原则

- 底盘专属逻辑放入 `chassis/`。
- 云台专属逻辑放入 `gimbal/`。
- 多个子系统共享的纯控制算法放入 `control/`。
- 硬件无关的数据类型和协议放入 `domain/`。
- STM32 外设读写只放入 `platform/`。
- 跨子系统编排只放入 `app/`。

## 安全模型

- 上电默认锁定。
- SwC 高档立即锁定。
- SwC 中档且四个主摇杆居中持续 1 秒后解锁。
- 遥控超时、failsafe、帧丢失立即清零。
- 任一模块反馈不新鲜时，该模块 PID 清零且电流清零。
- 俯仰目标有软件限位。
- 6623 和 GM6020 使用各自电流限幅。
