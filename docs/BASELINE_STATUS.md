# 基础框架完成状态

本文定义 `RM-Robot-Rust` 基础框架的稳定边界。后续功能应在这些边界之上扩展，
不应绕过安全门、直接从中断修改控制目标，或把具体子系统重新塞回通用目录。

## 已完成的基础功能

### 底盘

- 四个 M3508/C620 固定映射：左前 ID1、右前 ID2、左后 ID3、右后 ID4。
- CAN1 `0x201..0x204` 反馈解析与 `0x200` 群发电流控制。
- 普通轮差速运动学。
- 麦克纳姆轮 X 型运动学。
- SwB CH6 切换轮型，切换时清 PID 并零电流一个周期。
- 四路独立速度 PID。
- 四电机全在线安全门。
- 目标转速斜坡、单路电流斜坡、单路限幅和总电流预算。
- 普通轮/麦克纳姆轮双模式里程计。
- 反馈失联时停止里程计积分。

### 云台

- CAN2 RoboMaster 6623 偏航反馈专用解析。
- CAN2 GM6020 俯仰标准反馈解析。
- `0x1FF` 云台电流群发。
- 偏航和俯仰位置外环、速度内环级联控制。
- 6623 编码器差分测速和低通滤波。
- 俯仰软件角度限位。
- 云台反馈失联锁零。
- 失联重连时编码器重同步，避免速度和电流尖峰。
- 未配置机械零点和方向时云台默认锁零。

### 整车与平台

- FS-i6 + FS-A8S S.BUS 解码。
- SwC CH5 安全解锁、超时和 failsafe 锁车。
- 左摇杆控制底盘，右摇杆控制云台。
- 1 kHz 固定周期整车编排。
- CAN/USART 中断只采集数据，控制算法留在主循环。
- 无堆分配、定长数据结构。
- CAN、S.BUS、模块在线状态和控制周期诊断变量。
- 电机 20 ms 超时、控制漏拍锁零和约 500 ms 独立看门狗。
- 主机单元测试、Clippy 严格检查和 Cortex-M4F CI 构建。

## 装车后必须标定

以下项目属于具体车辆的调试参数，不代表基础框架缺失：

- 四个底盘电机实际方向。
- 轮子有效半径、轮距、轴距和减速比。
- 底盘 PID、单路电流限幅和总电流预算。
- 麦克纳姆轮安装方向与横移符号。
- 6623/GM6020 机械零位和方向。
- 俯仰机械限位。
- 云台两级 PID 与负载补偿。

## 稳定扩展点

- `src/chassis/`：底盘模式、功率模型、跟随控制。
- `src/gimbal/`：世界坐标控制、自动瞄准、云台标定。
- `src/estimation/`：IMU、姿态融合、轮速/视觉融合。
- `src/domain/`：跨模块数据类型和纯协议。
- `src/platform/`：硬件驱动和中断适配。
- `src/app/`：整车模式编排和跨子系统协调。

## 基线验收命令

```sh
make check
arm-none-eabi-size target/thumbv7em-none-eabihf/release/rm_robot
arm-none-eabi-readelf -h target/thumbv7em-none-eabihf/release/rm_robot
```

`make check` 必须同时通过格式检查、主机单元测试、Clippy 和 ARM 发布构建。
